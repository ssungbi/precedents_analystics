# 손사봇 판례/조정례 분석 스킬 사용 지침

당신은 사용자로부터 판례 PDF 파일 경로, 텍스트 내용, 또는 웹페이지 URL을 전달받았을 때 이 스킬을 가동하여 **직접 텍스트를 분석**하고, 후처리 스크립트를 호출해야 합니다.

## 단계 1: 분석 및 JSON 추출 (에이전트 직접 수행)
전달받은 판례 전문(또는 PDF 추출 텍스트)을 읽고, 당신의 훌륭한 추론 지능을 활용해 오직 아래의 JSON 포맷으로만 심층 분석 결과를 도출하십시오.
(JSON 데이터 외의 다른 부연 설명이나 마크다운 코드블록(` ```json `)을 포함하지 말고 오직 순수 JSON 문자열만 출력하는 것이 가장 좋습니다.)

```json
{
    "title": "사건번호_대상_핵심쟁점 (예: 2023나223773_뇌하수체_임상악성)",
    "case_no": "사건번호",
    "court": "법원 또는 기관명",
    "year": "선고연도",
    "date": "선고일/결정일 (YYYY-MM-DD)",
    "case_type": "사건유형",
    "tags": ["핵심질병또는담보", "핵심쟁점", "보험금분쟁"],
    
    "basic_info": "기본 정보 요약 (보험상품, 지급금액, 진단명, KCD 등 원문에서 확인된 것만)",
    "favorability": "유리, 불리, 또는 중립 (소비자/원고 입장에서의 유불리 최종 판정)",
    "favorability_reason": "위 유불리 판단의 핵심 이유 (1문장)",
    
    "core_issues": {
        "direct_issue": "실제 이 사건에서 다툰 문제 (사건 직접 쟁점)",
        "legal_issue": "약관/법리 쟁점 (진단확정, 면책기간, 고지의무 등)",
        "practical_issue": "유사 사건에서 어떤 자료를 확보해야 하는지에 대한 실무 확장 쟁점"
    },
    
    "fact_timeline": [
        "사실관계 1 (반드시 날짜순으로 정리: 계약 -> 사고 -> 진단 -> 치료 -> 청구 -> 거절 -> 소송)"
    ],
    "recognized_facts": [
        "이 사건에서 원문상 확인된 사실 (의학 사건의 경우 병리/임상/영상/소견 구분)"
    ],
    
    "court_decision": ["법원/기관의 판단"],
    "diagnosis_and_liability_date": "진단확정일 및 책임개시일 판단 내용 (있을 경우)",
    
    "acceptance_criteria": ["인정 요건 (원칙)"],
    "rejection_criteria": [
        "배척 요건 (단순 부정형이 아닌, 예외 상황이나 한계를 구체적으로 작성. 예: 조직학적 양성이면서 악성을 뒷받침할 자료가 부족한 경우)"
    ],
    
    "practical_points": ["실무상 추가로 확인할 자료 및 실무 적용 포인트"],
    "insurer_actual_claim": ["원문상 확인된 보험사 실제 주장"],
    "expected_rebuttals": ["유사 사건 시 보험사 예상 반론"],
    "counter_logic": ["그에 대한 반박 논리"],
    "cautions": ["이 판례를 실무에 적용할 때 원문에 없어서 쓰면 안 되는 내용이나 주의할 점"],
    
    "infographic_text": "인스타그램/오픈카톡용 1페이지 인포그래픽에 들어갈 아주 짧고 강렬한 핵심 요약 (글머리 기호 사용)",
    "blog_post": "네이버 블로그에 포스팅할 수 있는 형태의 상세 분석글 (마크다운 포맷)"
}
```

## 단계 2: 임시 파일로 저장
위 포맷에 맞게 분석된 JSON 데이터를 작업 디렉토리의 `scratch/agent_result.json` 파일로 저장하십시오.
(원본 텍스트도 `scratch/agent_raw_text.txt`로 저장해두면 옵시디언 원문 첨부에 도움이 됩니다.)

## 단계 3: 후처리 스크립트 실행
터미널(run_command) 도구를 열고, 이 스킬 폴더 내부에 있는 `scripts/cli_post_process.py`를 실행하여 옵시디언 저장, 이미지 생성, 깃허브 푸시를 자동 수행하십시오.
※ 실행 시 필요한 의존성(PyYAML, GitPython, Pillow 등)이 설치된 환경이어야 합니다.

```bash
# 에이전트의 현재 작업 환경(workspace)에 맞춰 스크립트 경로를 지정하세요.
# 예시:
python scripts/cli_post_process.py scratch/agent_result.json scratch/agent_raw_text.txt
```

## 단계 4: 결과 보고
스크립트 실행이 성공적으로 완료되면, 사용자에게 분석이 완료되었음을 브리핑하십시오.
- `core_issue`, `case_no` 등 핵심 내용 요약
- 중복 여부(스크립트 로그 참조) 및 저장 성공 여부
